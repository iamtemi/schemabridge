---
title: Known Limitations
---

SchemaBridge is intentionally small. These are the boundaries to know before relying on generated output.

## Exported Schemas Only

Folder conversion only scans exported schemas:

```ts
// Converted
export const userSchema = z.object({ id: z.string() });

// Ignored
const internalSchema = z.object({ id: z.string() });
```

This keeps conversion explicit and avoids a full TypeScript AST ownership model.

## Runtime-Only Zod Logic

`.transform()`, `.refine()`, `.superRefine()`, `.check()`, and `.preprocess()` are runtime behavior. SchemaBridge emits types from the input shape and warns when runtime-only effects are ignored.

```ts
const schema = z.number().transform((n) => n * 2);
// Emits a number type, not the transformed runtime result.
```

## Cross-File References

Imported schemas can be used when the module loads correctly. Generated files are still created per exported schema in the current scan.

If `extendedSchema` uses `baseSchema`, SchemaBridge may inline the nested shape inside `extendedSchema`. It will only create a separate top-level file for `baseSchema` if `baseSchema` is also exported and included in the conversion.

Use folder conversion when you want every exported schema in a tree:

```bash
schemabridge convert folder ./src --out ./generated --to pydantic
```

## Circular Schemas

Circular or deeply recursive schemas are not a good fit yet. Prefer one-way schema dependencies or convert the simpler boundary schemas first.

## TypeScript Loading

SchemaBridge loads trusted local schema files by executing them in Node.

- Run from the project root when schemas rely on project-level `tsconfig.json`.
- Use `--tsconfig <path>` for explicit path mapping.
- Only `baseUrl` and `paths` are used from `tsconfig.json`.
- Set `trustedInput: true` only for files you control.

## Output Naming

Nested object names are derived from field paths. If generated names are not what you want, split important nested schemas into exported top-level schemas.

Mutable defaults in Python use `Field(default_factory=...)` so generated Pydantic models avoid shared mutable state:

```python
tags: List[str] = Field(default_factory=list)
```

## Large Projects

Folder scanning is deterministic and works well for normal projects. For very large trees, skip directories you do not want scanned:

```ts
await convertFolder({
  sourceDir: './src',
  outDir: './generated',
  target: 'pydantic',
  trustedInput: true,
  ignore: ['node_modules', 'dist', 'tests'],
});
```

## Reporting Issues

If a schema shape should be supported but is not, open an issue with a minimal Zod example, the command/API call, and the generated output you expected.
